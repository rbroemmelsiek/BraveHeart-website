## CMS-Lite Structure Implemented

```text
content/braveheart/posts/          Markdown posts with YAML frontmatter
src/lib/frontmatter.ts             Frontmatter parser
src/lib/blog.ts                    import.meta.glob loader + queries
src/lib/markdownBody.tsx           Body renderer (headings, lists, links)
src/pages/BlogIndex.tsx            Public index (published only)
src/pages/BlogPost.tsx             Public post + 404 for unpublished slugs
src/pages/TopicArchive.tsx         Category/tag archive pages
scripts/generate-sitemap.mjs       Build-time sitemap including published posts
```

No database, backend CMS, authentication, admin UI, or form submission backend was added.

## Published vs Draft Behavior

| Behavior | Published | Draft / Review / Approved / Archived |
|---|---|---|
| Blog index listing | Yes | No |
| `/blog/:slug` render | Yes | Shows "Article Not Found" |
| Sitemap inclusion | Yes | No |
| SEO robots | From frontmatter (`index,follow` on sample) | Direct slug access uses `noindex,nofollow` |
| Related post links | Resolved if target is published | Unpublished slugs omitted |

Duplicate slugs throw at module load time.

## Taxonomy Behavior

- Categories and tags slugified via `taxonomySlug()` (lowercase, hyphenated).
- `/topics/:taxonomySlug` lists published posts where category or any tag matches.
- Example: published post category `Eligibility` → `/topics/eligibility`.
- Example: tag `public-safety` → `/topics/public-safety`.
- Topic archive pages are indexable but excluded from sitemap.

## SEO Behavior

- `SEOHead` calls `getBlogSEO(pathname)` before static Phase 1 SEO.
- Blog index uses fixed metadata in `src/lib/blog.ts`.
- Published posts use frontmatter `seoTitle`, `seoDescription`, `robots`, `canonicalUrl`, and share/OG image filenames.
- Blog posts set `og:type` to `article`.
- Unpublished slug URLs and unknown topic archives use `noindex,nofollow`.

5. **JSON-LD Article schema** — Not implemented (deferred).
6. **Navbar blog link** — Footer link only to preserve Phase 1 nav density.
7. **SPA crawling** — Blog metadata requires JavaScript execution unless prerender added later.
8. **Approved/archived status handling** — `approved` treated like draft for public visibility until explicitly published.
