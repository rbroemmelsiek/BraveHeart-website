## SEO Metadata Mechanism

**Approach:** Lightweight custom `SEOHead` component using `useEffect` to upsert `document.title`, `<meta>`, and `<link rel="canonical">` tags. No new npm dependency.

**Why not react-helmet-async:** Avoids bundle increase; existing SPA pattern is sufficient for Phase 1. Static `index.html` provides home fallback for non-JS crawlers.

**Flow:**

1. `src/config/seo.ts` defines metadata for all 10 Phase 1 routes.
2. `AppShell` reads `useLocation().pathname` and passes it to `SEOHead`.
3. `SEOHead` updates head tags on every route change.
4. Unknown/legacy redirect paths fall back to home metadata briefly before redirect.
