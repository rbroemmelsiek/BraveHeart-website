# 02. Brave Heart Content Pages

⚠️ **Medium** — Inserts approved page content while maintaining layout quality.

## Objective

Implement approved Brave Heart content across the Phase 1 route set.

## Repo Inspection Preconditions (2026-06-21)

- **Depends on plan 01** — target routes and stub pages must exist.
- `docs/braveheart/normalized-content/*.md` files contain **target H1, section outlines, and draft status only** — body copy is not implementation-ready.
- `docs/braveheart/content-map.md` marks all sections as **Draft**.
- Legal normalized content requires compliance review before publication (`legal-and-compliance-notes.md`).
- Reuse listing page **layout patterns** (hero sections, editorial grids, `.reveal` animations, legal numbered sections) — do not rebuild page architecture.
- Home page is ~400 lines monolithic; other listing pages similar — expect inline sections unless repetition warrants extraction.

## Scope

- Use `docs/braveheart/content-map.md` as editorial placement map.
- Use `docs/braveheart/normalized-content/*.md` as content source **after approval**.
- Implement page sections preserving responsive layout and existing CSS utilities.
- Add CTA placeholder blocks where specified (full wiring in plan 04).
- Replace all listing-site property copy on public pages.

## Out of Scope

- New backend forms.
- Full verification logic.
- Blog/CMS (plan 06).
- Unapproved legal claim expansion.
- Final WebP image insertion (plan 03 — use approved alt text + placeholder or temporary PNG paths if needed).
- SEO metadata injection (plan 05).

## Files Likely Affected

- `src/pages/Home.tsx` (full rewrite)
- `src/pages/OurStory.tsx`, `WhoWeServe.tsx`, `SavingsComparison.tsx`, `ServiceLines.tsx`, `OurGuarantee.tsx`, `CaseStudies.tsx`
- `src/pages/PrivacyPolicy.tsx`, `TermsOfService.tsx`, `Accessibility.tsx` (copy replacement; preserve layout pattern)
- `docs/braveheart/content-map.md` (status updates)
- `.agent/validation/braveheart-site-validation.md`
- `PROGRESS.md`

## Validation

- Each route has route-specific content (no shared generic placeholder across pages).
- Exactly one H1 per page matching `seo-publication-map.md` where approved.
- No old listing-site property copy remains (grep for "1622", "Burbank", "Oak St", "Karen Van Ness", "tour scheduling").
- CTAs render as buttons/links (behavior validated in plan 04).
- Internal links between pages resolve.
- Mobile readability passes visual inspection at 375px and 768px widths.
- Legal pages flagged `needs-review` until owner/compliance sign-off.
- `npm run build` passes.
- `npm run lint` passes.

## Rollback Notes

Revert page file changes per route. Plan 01 stubs remain if individual pages fail validation. Do not revert Navbar/Footer route map.

## Risks Before Execution

- **Not single-pass feasible for full approved copy** — normalized content is outline-only; expect two passes (structure + copy) or blocked until content approval.
- Implementing manifest/marketing percentages without legal review violates CLAUDE.md compliance rules.
- Large monolithic page files increase regression risk — validate each route independently.
- Image placeholders may 404 until plan 03 — document temporary paths.

## Approval Gate

Stop before final asset pass if images are not approved. Stop legal pages if compliance review is incomplete.
