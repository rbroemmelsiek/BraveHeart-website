# 05. Brave Heart SEO and Publication Controls

⚠️ **Medium** — Adds SEO metadata, robots, sitemap, and publication checks.

## Objective

Prepare the Phase 1 static site for controlled publication.

## Repo Inspection Preconditions (2026-06-21)

- **No per-route metadata mechanism exists** — only static `<title>1622 W. Oak St.</title>` in `index.html`.
- `public/robots.txt` and `public/sitemap.xml` **do not exist**.
- `docs/braveheart/seo-publication-map.md` has draft metadata for all 10 routes (placeholder domain `https://braveheartfirstresponders.com`).
- OG image references point to WebP filenames from plan 03 — may not exist until conversion completes.
- **Depends on plans 01–02** for routes and H1 copy; **coordinates with plan 03** for OG image paths.
- React 19 SPA — metadata likely needs a small helper (e.g. `react-helmet-async`, custom `SEOHead` with `useEffect` + `document.head`, or `vite-plugin-html` per-route — choose minimal approach matching repo conventions).

## Scope

- Implement a lightweight per-route metadata mechanism.
- Add title, meta description, canonical URL, and Open Graph tags for all Phase 1 routes per `seo-publication-map.md`.
- Create `public/robots.txt` (staging vs production rules documented).
- Create `public/sitemap.xml` with all Phase 1 public routes.
- Validate internal links across nav, footer, and page cross-links.
- Update `seo-publication-map.md` status fields.

## Out of Scope

- Advanced analytics or tag managers.
- Paid search setup.
- JSON-LD schema beyond basic OG unless explicitly approved.
- Blog/CMS sitemap entries (plan 06).
- Production domain finalization (use placeholder domain with documented swap step).

## Files Likely Affected

- **New:** `src/components/SEOHead.tsx` (or equivalent metadata helper)
- `src/pages/*.tsx` (metadata props or calls per route)
- `public/robots.txt`
- `public/sitemap.xml`
- `index.html` (default/fallback meta)
- `docs/braveheart/seo-publication-map.md`
- `docs/braveheart/seo-and-publication-checklist.md` (checklist pass)
- `.agent/validation/braveheart-site-validation.md`
- `PROGRESS.md`

## Tasks

1. Choose and implement minimal metadata approach (document choice in validation report).
2. Add unique `<title>` and `<meta name="description">` per Phase 1 route from `seo-publication-map.md`.
3. Add canonical URL per route using placeholder domain (env var or config constant for easy swap).
4. Add Open Graph tags (`og:title`, `og:description`, `og:image`, `og:url`) per route.
5. Create `public/robots.txt` — default `Allow: /` with sitemap reference; document staging `noindex` variant.
6. Create `public/sitemap.xml` listing all 10 Phase 1 routes with placeholder absolute URLs.
7. Verify exactly one H1 per page matches seo map (cross-check plan 02).
8. Audit internal links — no references to old listing routes or slugs.
9. Update `seo-publication-map.md` implementation status.

## Validation

- Unique SEO title per public route (view page source or devtools).
- Unique meta description per public route.
- Exactly one H1 per page (unchanged from plan 02).
- `public/robots.txt` exists and references sitemap URL.
- `public/sitemap.xml` includes all 10 Phase 1 routes with correct paths.
- OG image paths resolve (or documented as pending if plan 03 incomplete).
- No accidental `noindex` on production-intended pages.
- Canonical URLs use consistent placeholder domain.
- `npm run build` passes; inspect `dist/` for robots/sitemap copy.
- `npm run lint` passes.

## Rollback Notes

Remove SEOHead usage from pages. Delete `robots.txt` and `sitemap.xml`. Restore single static title in `index.html`.

## Risks Before Execution

- SPA metadata may not appear in all crawlers without SSR/prerender — document limitation if static-only.
- Placeholder domain in sitemap/canonical must be swapped before launch or search indexing will be wrong.
- OG images 404 if plan 03 not complete — validate paths or use logo fallback.
- Adding a new dependency (helmet) increases bundle — prefer lightweight custom helper if sufficient.

## Approval Gate

Stop before deployment. Confirm production domain with owner before swapping placeholder URLs.
