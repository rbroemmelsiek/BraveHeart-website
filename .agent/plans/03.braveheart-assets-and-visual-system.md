# 03. Brave Heart Assets and Visual System

⚠️ **Medium** — Converts images and applies brand tone without destroying working styling.

## Objective

Convert Brave Heart image assets to WebP, reorganize to the production path, update brand tokens, and insert approved images.

## Repo Inspection Preconditions (2026-06-21)

- **Source images are at `public/assets/`**, not in `inbox/`. Inbox (`public/assets/images/braveheart/inbox/`) contains only `.gitkeep`.
- On disk: 16 PNG + 3 WebP Brave Heart assets; 1 listing legacy WebP (`Karen2-250px.webp`) to retire.
- Planned production path `public/assets/images/braveheart/` is empty (except inbox).
- Most assets are still PNG; `asset-inventory.md` lists WebP target names — conversion not started.
- `image-conversion-report.md` is empty (TBD row only).
- Current theme in `src/index.css` is **listing-site light luxury** (warm beige, steel blue `#355f86`) — Brave Heart target is deep navy + gold per `brand-and-style-guide.md`.
- **Depends on plan 02 page structure** (or plan 01 stubs) for image insertion targets.

## Scope

- Inventory and reconcile all images at `public/assets/` against asset inventory.
- Copy/move sources through inbox workflow if required by governance, then convert PNG/JPG/JPEG to WebP.
- Output production WebP files to `public/assets/images/braveheart/` preserving dimensions.
- Update `asset-inventory.md` and `image-conversion-report.md`.
- Remap `@theme` color tokens and utility classes in `src/index.css` for Brave Heart (navy/gold/dark glass).
- Insert approved images into mapped page sections with alt text from inventory/manifest.
- Normalize filenames (replace spaces; match SEO names in inventory).

## Out of Scope

- Full redesign or new component library.
- Image upscaling.
- Deleting originals without approval.
- Listing-site image restoration (retire, do not recover missing root `/ *.webp` files).
- Favicon/apple-touch-icon generation unless explicitly requested.
