# Brave Heart Quality Gates

## Purpose

Defines gates that must pass before each phase is considered complete.

## Gate 1: Repo Inspection

- Existing routes documented.
- Existing components documented.
- Existing style tokens documented.
- Safe edit files identified.
- Risks listed.

## Gate 2: Source Normalization

- Content sources identified.
- Target routes mapped.
- Asset inventory created.
- Misnamed legacy sources documented.
- No code changed without approval.

## Gate 3: Route Shell

- All Phase 1 routes render.
- Navbar links work.
- Footer links work.
- Old listing-site routes removed or redirected.
- No old property copy remains in visible navigation.

## Gate 4: Content Pages

- Each page has approved copy.
- Page sections match content map.
- CTAs appear where intended.
- Legal pages are present.

## Gate 5: Assets

- Images converted to WebP where practical.
- Image paths work.
- Alt text present.
- Asset inventory updated.
- Mobile layout stable.

## Gate 6: SEO / Publication

- Metadata exists for all routes.
- robots.txt exists.
- sitemap exists or generator documented.
- Open Graph images exist.
- Canonicals are correct.
- Build passes.

## Gate 7: Phase 2 CMS-Lite

- Blog index renders.
- Post route renders.
- Category/tag pages render if implemented.
- Draft/review/published status works.
- Sitemap includes published posts only.
